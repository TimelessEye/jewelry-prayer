# 용문교회 유치부 기도보석 챌린지 앱 기획서

작성일: 2026-06-21  
운영 기간: 2026-06-22 ~ 2026-07-11, 총 20일  
대상: 용문교회 유치부 부모님과 교사  
배포 방식: GitHub + Vercel 배포, PWA 설치 지원  
작업 원칙: 실제 개발은 `dev` 브랜치를 만들어 진행한다.

## 1. 프로젝트 개요

기도보석 챌린지는 부모님과 교사가 20일 동안 매일 다음세대를 위해 기도하고, 기도 완료 후 날짜별 보석을 수집하는 모바일 우선 웹앱이다.

앱은 링크로 접속할 수 있고, 원하는 사용자는 홈 화면에 추가하여 앱처럼 설치할 수 있다. 부모와 교사는 같은 기도문을 보고 같은 보석 수집 흐름을 사용하되, 관리자 통계에서는 부모와 교사를 분리해서 볼 수 있어야 한다.

핵심 경험은 다음과 같다.

- 오늘의 기도문을 쉽게 시작한다.
- 기도 완료 후 반짝이는 보석을 수집하는 느낌을 받는다.
- 20칸 보석 수집장에서 자신의 진행 상황을 확인한다.
- 놓친 과거 날짜의 빈 보석 자리를 눌러 다시 기도할 수 있다.
- 관리자는 전체 참여 현황과 완주율을 한눈에 확인한다.

## 2. 확정된 주요 방향

- 첫 화면에는 `부모로 참여하기`와 `교사로 참여하기` 두 개의 큰 선택지가 뜬다.
- 부모는 자녀 이름을 선택하고 `맘 / 대디`를 선택한다.
- 부모 표시명 예시는 `은우 맘`, `선예·선하 대디`이다.
- 기존 유치부 명단에 있는 자녀는 명단에서 선택한다.
- 명단에 없는 자녀는 직접 입력할 수 있다.
- 명단 외 자녀 참여자는 일반 기능은 동일하게 사용하지만, 관리자 통계에서는 공식 명단 통계와 분리한다.
- 같은 자녀에 대해 맘과 대디가 각각 참여할 수 있다.
- 관리자는 부모 둘 다 참여한 가정을 확인할 수 있다.
- 교사는 교사 명단에서 본인을 선택한다.
- 교사 표시명은 `김현진 선생님` 형식이다.
- 부모와 교사는 동일한 기도문, 동일한 기도 완료 흐름, 동일한 보석 수집 UX를 사용한다.
- 부모 통계와 교사 통계는 관리자 화면에서 분리한다.
- 기도문은 날짜별로 PNG 이미지 3장 고정이다.
- 기도문은 관리자가 하루 전에 업로드할 수 있지만, 사용자에게는 해당 날짜 0시부터 공개된다.
- 전체 기도문 보기에서는 오늘과 과거에 공개된 기도문만 볼 수 있다.
- 미래 기도문은 열리지 않는다.
- 놓친 과거 날짜의 빈 보석을 누르면 해당 날짜의 기도문으로 이동한다.
- 기도 완료 후 해당 날짜의 보석을 받을 수 있다.
- 관리자 접근은 오른쪽 위 설정에서 비밀번호를 입력하는 숨긴 기능으로 제공한다.
- 관리자 비밀번호는 코드에 하드코딩하지 않는다.

## 3. 사용자 역할

### 3.1 부모 참여자

부모는 자녀를 위해 20일 동안 기도하는 사용자이다.

부모 등록 방식:

1. 첫 화면에서 `부모로 참여하기` 선택
2. 기존 자녀 명단에서 자녀 선택
3. 자녀가 여러 명이면 복수 선택
4. 명단에 없는 경우 직접 입력
5. `맘 / 대디` 중 하나 선택
6. 표시명 자동 생성
7. 다음 접속부터 자동 진입

예시:

- `은우 맘`
- `은우 대디`
- `선예·선하 맘`
- `선예·선하 대디`

다음 방문 UX:

- 앱이 저장된 참여자 정보를 찾으면 `은우 맘으로 계속하기`를 보여준다.
- 사용자가 원하면 `다른 이름으로 시작하기`를 선택할 수 있다.
- 자동 로그인 정보는 브라우저 localStorage에 저장하되, 실제 참여 기록은 Supabase에 저장한다.

### 3.2 교사 참여자

교사는 유치부 다음세대를 위해 같은 기도문으로 기도하는 사용자이다.

교사 등록 방식:

1. 첫 화면에서 `교사로 참여하기` 선택
2. 교사 명단에서 본인 선택
3. 표시명은 `이름 + 선생님`으로 자동 생성
4. 다음 접속부터 자동 진입

교사 명단은 앱에서 가나다순으로 보여준다. 기본 명단은 다음과 같다.

- 강순진
- 김현진
- 박옥희
- 성유리
- 신미경
- 오선녀
- 윤정아
- 이린자
- 이선예
- 정은지
- 조문경
- 조선영
- 홍명환

교사는 부모와 같은 UX를 사용한다.

- 오늘의 기도 시작하기
- 전체 기도문 보기
- 기도 마치기
- 보석 수집
- 보석 수집장
- 놓친 날짜 기도하기
- 완주 공유 이미지 만들기

관리자 화면에서는 교사 통계를 부모 통계와 따로 보여준다.

### 3.3 관리자

관리자는 앱 운영자이다.

관리자 진입:

- 앱 오른쪽 위 설정 버튼 선택
- 관리자 비밀번호 입력
- 성공 시 관리자 대시보드 진입

관리자 기능:

- 날짜별 기도문 이미지 3장 업로드 및 교체
- 오늘까지 공개된 기도문 상태 확인
- 부모 참여 통계 확인
- 교사 참여 통계 확인
- 명단 외 참여자 확인
- 각 참여자의 진행률 확인
- 부모 둘 다 참여한 가정 확인
- 전체 완주율 확인
- 교사 완주자 확인
- 필요 시 보석/공유 이미지 템플릿 관리

관리자 비밀번호 보안 원칙:

- 비밀번호를 React 코드나 `VITE_` 환경변수에 넣지 않는다.
- 기존 가정예배 확인 앱처럼 Supabase Auth 관리자 계정으로 검증하는 방식을 우선 검토한다.
- Vercel 서버 환경변수로 검증하는 경우에는 서버 전용 변수만 사용한다.

## 4. 핵심 사용자 흐름

### 4.1 첫 화면

첫 접속 시 화면에는 두 개의 큰 선택 카드가 나온다.

상단 카드:

- 부모로 참여하기

하단 카드:

- 교사로 참여하기

화면에는 앱 이름을 표시한다.

- 메인 타이틀: `기도보석 챌린지`
- 보조 표기: `용문교회 유치부`

`용문교회 유치부`는 너무 크게 강조하지 않고, 화면 한쪽이나 타이틀 아래 작은 위치에 자연스럽게 배치한다.

### 4.2 부모 등록 흐름

부모가 `부모로 참여하기`를 누르면 자녀 선택 화면으로 이동한다.

자녀 선택:

- 기존 유치부 명단에서 선택
- 자녀가 2명 이상이면 복수 선택 가능
- 명단에 없으면 `명단에 없는 자녀 직접 입력` 사용

보호자 선택:

- `맘`
- `대디`

표시명 생성:

- 자녀 1명: `은우 맘`
- 자녀 2명: `선예·선하 대디`
- 직접 입력 자녀도 같은 규칙 적용

등록 완료 후:

- 참여자 정보가 Supabase에 저장된다.
- 브라우저에는 해당 참여자 ID와 자동 로그인용 토큰을 저장한다.
- 바로 홈 화면으로 이동한다.

### 4.3 교사 등록 흐름

교사가 `교사로 참여하기`를 누르면 교사 선택 화면으로 이동한다.

교사 선택:

- 가나다순 명단에서 이름 선택
- 선택 후 `김현진 선생님` 형식으로 표시명 생성

등록 완료 후:

- 교사 참여자 정보가 Supabase에 저장된다.
- 브라우저에는 자동 로그인용 정보를 저장한다.
- 바로 홈 화면으로 이동한다.

### 4.4 참여자 홈

등록 후 홈 화면에는 사용자가 바로 기도할 수 있는 구성이 나온다.

주요 요소:

- 오늘 날짜
- 참여자 표시명
- 오늘의 기도 상태
- `오늘의 기도 시작하기` 버튼
- `보석 수집장 보기` 버튼
- `전체 기도문 보기` 버튼
- 오른쪽 위 설정 버튼

오늘 기도를 이미 완료한 경우:

- `오늘 기도보석을 수집했어요` 상태 표시
- `보석 수집장 보기`를 더 강조
- 기도문은 다시 볼 수 있음

오늘 기도문이 아직 공개 전인 경우:

- `기도문은 당일 자정에 열립니다.`
- 관리자가 이미 업로드했더라도 날짜가 되기 전에는 사용자에게 보이지 않는다.

오늘 기도문이 아직 업로드되지 않은 경우:

- `오늘 기도문을 준비 중입니다.`
- 관리자가 업로드하면 공개 시간 이후 자동으로 보인다.

### 4.5 오늘의 기도 보기

`오늘의 기도 시작하기`를 누르면 오늘 날짜의 기도문 화면으로 이동한다.

기도문 구성은 하루 3장 고정이다.

1. 기도문
2. 관련 말씀과 선포/축복간구 권유 문구
3. 성경학교 기도제목 나눔

화면 구성:

- 상단 날짜와 진행 표시
- PNG 이미지 3장 세로 보기 또는 슬라이드 보기
- 마지막에 `기도 마치기` 버튼

모바일에서 이미지가 너무 작지 않도록 화면 폭에 맞게 표시한다. 확대해서 볼 필요가 있으면 탭 확대 또는 핀치 줌이 자연스럽게 가능해야 한다.

### 4.6 전체 기도문 보기

홈의 `전체 기도문 보기`를 누르면 공개된 기도문 목록으로 이동한다.

표시 규칙:

- 오늘 기도문 포함
- 과거 공개 기도문 포함
- 미래 기도문 숨김
- 업로드는 되었지만 아직 공개일이 안 된 기도문 숨김

목록 예시:

- 6월 22일 기도문
- 6월 23일 기도문
- 6월 24일 기도문

각 날짜를 누르면 해당 날짜의 기도문 3장을 볼 수 있다.

이미 기도 완료한 날짜:

- 완료 상태와 수집된 보석 표시

아직 기도하지 않은 과거 날짜:

- 기도문을 보고 완료할 수 있음
- 완료 후 해당 날짜 보석 수집 가능

### 4.7 기도 완료와 보석 수집 연출

기도 완료 후 보석을 수집하는 경험은 이 앱의 핵심 재미 요소이므로 v1 필수 UX로 구현한다.

흐름:

1. 사용자가 `기도 마치기` 클릭
2. 화면이 살짝 어두워짐
3. 반짝이는 보석 팝업 등장
4. 문구 표시
   - `오늘의 기도보석을 발견했어요.`
   - `기도보석을 수집하시겠습니까?`
5. 사용자가 `수집하기` 클릭
6. 보석이 빛나며 작아지거나 이동하는 애니메이션 실행
7. 보석 수집장으로 이동
8. 해당 날짜 칸에 보석이 반짝이며 자리 잡음
9. 완료 문구 표시
   - `6월 22일 기도보석을 수집했어요.`

놓친 날짜 기도도 같은 흐름을 사용한다.

- 지난 날짜 빈 자리 클릭
- 해당 날짜 기도문 보기
- `기도 마치기`
- 그 날짜의 보석 팝업
- `수집하기`
- 수집장 해당 날짜에 보석 채워짐

### 4.8 보석 수집장

보석 수집장은 20일 전체 진행을 한눈에 보여주는 핵심 화면이다.

구성:

- 20개 보석 자리
- 각 자리는 날짜와 연결
- 수집한 날짜는 색이 다른 보석 표시
- 아직 수집하지 않은 날짜는 빈 자리 표시
- 미래 날짜는 잠긴 상태 또는 흐린 상태 표시

상단 안내 문구:

`아직 수집하지 못한 보석 자리를 누르면 지난 기도문으로 이동합니다.`

동작 규칙:

- 수집 완료 자리: 완료 상태 확인
- 과거 또는 오늘의 미수집 자리: 해당 날짜 기도문으로 이동
- 미래 날짜 자리: 열리지 않음
- 미래 날짜를 누르면 `기도문은 당일 자정에 열립니다.` 안내

보석 색상:

- 20일 각각 다른 색을 사용한다.
- 모양은 같아도 된다.
- 사용자가 보석 PNG 1개를 제공하면, 앱에서 Canvas 또는 CSS 필터로 20가지 색상 변형을 만든다.
- 개발 시 20색 팔레트를 정해 날짜별로 고정한다.

### 4.9 완주 공유

20개 보석을 모두 수집하면 완주 축하 화면이 열린다.

완주 카드는 일반 메뉴로 노출하지 않는다. 사용자가 20번째 보석을 수집하는 순간 시스템이 자동으로 완주 축하 화면을 열고, 그때 공유 이미지를 생성한다.

공통 흐름:

1. 완주 축하 화면 표시
2. 공유 이미지 자동 생성
3. `공유하기` 버튼 제공
4. 지원되는 기기에서는 네이티브 공유창 호출
5. 공유가 제한되는 환경에서는 이미지 저장과 문구 복사 제공

마지막 날 엣지케이스:

- 2026년 7월 11일 이후에도 아직 수집하지 못한 보석이 있으면 홈에서 남은 기도를 이어갈 수 있는 안내를 보여준다.
- 남은 기도가 20일차 하나뿐이면 `마지막 남은 오늘의 기도에 참여하시겠습니까?`라고 안내하고 20일차 기도문으로 보낸다.
- 남은 기도가 여러 개이면 `남은 기도를 완주하시겠습니까?`라고 안내하고 보석 수집장으로 보낸다.
- 마지막 보석이 20일차든 놓친 과거 날짜든, 총 20개가 모두 모이는 순간에만 완주 카드를 제공한다.

부모 공유 이미지:

- 배경: 예쁜 유리병 안에 보석이 담긴 이미지
- 텍스트는 앱에서 자동 합성
- 예시 문구:
  - `은우 맘이 기도보석 챌린지를 완주했습니다.`
  - `축하합니다!`

교사 공유 이미지:

- 교사는 완주 시 13종의 기념 보석 중 아직 배정되지 않은 보석 1개를 무작위로 뽑아 받는다.
- 보석은 한 번 배정되면 목록에서 소거되어 다른 교사에게 다시 배정되지 않는다.
- 13명의 교사가 모두 완주하면 마지막 교사는 마지막 남은 보석을 받는다.
- 배정된 보석의 이미지, 이름, 설명을 사용해 교사용 완주 보석카드를 생성한다.
- 텍스트는 앱에서 자동 합성
- 확정 문구:
  - `기도보석 챌린지 완주를 축하합니다.`
  - `김현진 선생님은 유치부의 보석이에요!`
- 교사가 완주 화면에 다시 들어오면 새로 뽑지 않고 이미 배정된 보석을 다시 보여준다.

공유 기술:

- HTML Canvas로 배경 이미지 위에 이름과 문구를 합성한다.
- 결과물을 PNG 파일로 만든다.
- Web Share API를 지원하면 이미지 파일 공유를 시도한다.
- 카카오톡, 브라우저, iOS/Android 환경에 따라 파일 공유가 막히면 fallback을 제공한다.

## 6. 관리자 기능 상세

### 6.1 기도문 관리

관리자는 날짜별로 PNG 이미지 3장을 업로드한다.

각 날짜의 이미지 슬롯:

1. 기도문
2. 관련 말씀과 선포/축복간구 권유
3. 성경학교 기도제목 나눔

관리 화면 요구사항:

- 날짜별 업로드 상태 표시
- 각 날짜마다 3장 모두 업로드됐는지 확인
- 이미지 미리보기
- 이미지 교체
- 공개 예정 시간 표시
- 오늘 공개 여부 표시

### 6.2 부모 통계

관리자가 보고 싶은 부모 통계:

- 등록한 부모 참여자 수
- 오늘 기도 완료 부모 수
- 날짜별 부모 완료 수
- 부모별 진행률
- 부모별 완료 날짜 수
- 20일 완주 부모 수
- 부모 참여자 전체 완주율
- 일평균 부모 참여자 수
- 부모 둘 다 참여한 가정 목록
- 명단 외 자녀 참여자 목록

통계 분리 기준:

- 공식 명단 자녀를 선택한 부모는 공식 통계에 포함한다.
- 직접 입력한 자녀는 `명단 외 참여자`로 따로 보여준다.
- 명단 외 참여자는 전체 사용량 파악에는 보이지만, 반별 공식 통계에는 섞지 않는다.

부모 둘 다 참여한 가정:

- 같은 자녀 조합에 대해 `맘`과 `대디` 참여자가 모두 있으면 표시한다.
- 예: `은우 가정: 맘 14/20, 대디 12/20`
- 예: `선예·선하 가정: 맘 20/20, 대디 20/20`

### 6.3 교사 통계

관리자가 보고 싶은 교사 통계:

- 전체 교사 수
- 등록한 교사 수

- 날짜별 교사 완료 수
- 교사별 진행률
- 20일 완주 교사 수
- 교사 전체 완주율
- 일평균 교사 참여자 수

- 아직 배정되지 않은 교사용 보석 수

교사 전체 수의 기본값은 13명이다.

관리자 첫 화면에는 한눈에 보이는 요약 카드가 필요하다.

상세 표:

## 7. 데이터 저장 설계

백엔드는 Supabase를 사용한다.

저장 대상:

- 공식 자녀 명단
- 교사 명단
- 참여자 정보
- 참여자의 자녀 연결 정보
- 날짜별 기도문 이미지
- 날짜별 기도 완료 및 보석 수집 기록
- 공유 이미지 템플릿 정보

추천 테이블 구조:

### 7.1 `classes`

유치부 반 정보.

필드:

- `id`
- `name`
- `sort_order`
- `created_at`

기존 가정예배 확인 앱의 반 명단을 참고한다.

### 7.2 `students`

공식 자녀 명단.

필드:

- `id`
- `class_id`
- `name`
- `sort_order`
- `active`
- `created_at`

기존 `D:\06_project\home-worship-report-app`의 Supabase seed 데이터를 참고해 초기 명단을 가져올 수 있다.

### 7.3 `teachers`

교사 명단.

필드:

- `id`
- `name`
- `sort_order`
- `active`
- `created_at`

초기 명단은 이 기획서의 교사 명단을 사용한다.

### 7.4 `participants`

실제 챌린지 참여자.

필드:

- `id`
- `type`: `parent` 또는 `teacher`
- `display_name`
- `guardian_role`: 부모일 때 `mom` 또는 `daddy`
- `teacher_id`: 교사일 때 연결
- `source`: `official` 또는 `custom`
- `household_key`
- `access_token_hash`
- `created_at`
- `last_seen_at`

설명:

- 부모와 교사를 같은 참여자 테이블로 관리하면 기도 완료 기록을 공통 처리할 수 있다.
- 자동 로그인은 브라우저에 저장한 토큰과 DB의 토큰 해시로 검증한다.
- 같은 자녀 조합의 맘/대디를 찾기 위해 `household_key`를 사용한다.

### 7.5 `participant_children`

부모 참여자와 자녀 연결.

필드:

- `id`
- `participant_id`
- `student_id`: 공식 명단 자녀일 때 사용
- `custom_child_name`: 명단 외 직접 입력일 때 사용
- `created_at`

설명:

- 자녀가 여러 명이면 여러 행으로 저장한다.
- 공식 명단 자녀와 직접 입력 자녀를 구분한다.

### 7.6 `prayer_days`

20일 기도 일정.

필드:

- `id`
- `day_index`: 1~20
- `prayer_date`
- `publish_at`
- `title`
- `created_at`

설명:

- `publish_at`은 한국 시간 기준 해당 날짜 00:00으로 설정한다.
- 사용자에게 노출할 때 현재 시간이 `publish_at` 이상인지 확인한다.

### 7.7 `prayer_images`

날짜별 기도문 이미지.

필드:

- `id`
- `prayer_day_id`
- `slot`: 1, 2, 3
- `storage_path`
- `uploaded_at`
- `uploaded_by`

설명:

- 하루 3장 고정이다.
- 관리자가 이미지를 교체할 수 있다.

### 7.8 `prayer_completions`

기도 완료와 보석 수집 기록.

필드:

- `id`
- `participant_id`
- `prayer_day_id`
- `completed_at`
- `collected_at`
- `created_at`

제약:

- `participant_id + prayer_day_id`는 유니크
- 같은 날짜를 여러 번 눌러도 중복 기록되지 않는다.

설명:

- 기도 완료와 보석 수집은 사실상 같은 흐름이지만, 팝업에서 `수집하기`를 누른 시점을 `collected_at`으로 저장한다.

### 7.9 `share_templates`

완주 공유 이미지 템플릿.

필드:

- `id`
- `type`: `parent` 또는 `teacher`
- `storage_path`
- `text_area_json`
- `created_at`
- `updated_at`

설명:

- 부모용 유리병 보석 이미지와 교사용 보석카드 템플릿을 분리한다.
- 텍스트 위치와 폰트 크기 조정값을 저장할 수 있다.

### 7.10 `teacher_completion_gems`

교사 완주 시 중복 없이 배정할 13종 기념 보석 목록.

필드:

- `id`
- `sort_order`: 1~13
- `slug`
- `name_ko`
- `name_en`
- `description`
- `image_path`
- `active`
- `created_at`

설명:

- 초기 데이터는 `public/images/teacher/gem-manifest.json`을 기준으로 seed 한다.
- 교사용 보석 이미지는 `public/images/teacher/gems` 폴더에 정리되어 있다.

### 7.11 `teacher_gem_assignments`

완주한 교사에게 배정된 기념 보석 기록.

필드:

- `id`
- `participant_id`
- `teacher_completion_gem_id`
- `assigned_at`

제약:

- `participant_id`는 유니크여야 한다.
- `teacher_completion_gem_id`도 유니크여야 한다.

설명:

- 한 교사는 보석 1개만 받는다.
- 한 보석은 한 교사에게만 배정된다.
- 동시 완주 시 중복 배정을 막기 위해 DB 함수 또는 트랜잭션으로 남은 보석 1개를 안전하게 뽑는다.
- 추천 구현은 `assign_teacher_completion_gem(participant_id)` RPC를 만들고, 아직 배정되지 않은 보석 중 1개를 무작위로 선택해 insert 하는 방식이다.

## 10. 필요한 이미지 자산

사용자가 직접 줄 수 있는 이미지와 앱이 자동 생성할 이미지를 구분한다.

### 10.1 사용자가 준비하면 좋은 이미지

1. 보석 수집장 배경 이미지
   - 20개 보석 자리를 올리기 좋은 그림
   - 예: 보석함, 수집판, 따뜻한 챌린지 보드

2. 기본 보석 PNG 1개
   - 투명 배경
   - 정면에서 잘 보이는 보석
   - 앱이 20가지 색상으로 변형

3. 부모 완주 공유용 배경 이미지
   - 예쁜 유리병에 보석이 담긴 이미지
   - 이름과 문구가 들어갈 여백 필요

4. 교사 완주 보석 이미지 13종
   - 현재 `public/images/teacher/gems`에 개별 PNG 13개가 준비되어 있다.
   - 설명과 매핑은 `public/images/teacher/gem-manifest.json`을 사용한다.
   - 완주 교사에게 중복 없이 하나씩 배정한다.

5. 교사 완주 공유카드 샘플
   - 현재 `public/images/teacher/completion-card-sample.png`가 준비되어 있다.
   - 실제 공유 이미지는 선택된 보석, 보석 설명, 교사 이름을 합성해서 만든다.

6. 앱 아이콘용 이미지
   - 정사각형
   - 보석과 기도 챌린지 분위기

### 10.2 앱이 자동 처리할 이미지 작업

- 기본 보석 PNG를 20가지 색상으로 변형
- 보석 수집장에 날짜별 보석 배치
- 완주 공유 이미지에 사용자 이름과 문구 합성
- 부모/교사 공유 문구 분기
- 교사 완주 시 13종 보석 중 남은 보석 1개를 무작위 배정
- 교사용 완주 보석카드에 배정된 보석 이미지와 설명 합성

### 10.3 권장 이미지 규격

기도문 PNG:

- 세로형
- 권장: 1080px 폭
- 하루 3장 고정
- 모바일에서 읽기 쉬운 큰 글씨

## 13. 반응형 UI 기준

앱은 모바일 우선 반응형으로 구현한다. 부모와 교사는 대부분 휴대폰으로 접속하므로, 작은 모바일 화면에서도 모든 핵심 기능을 불편 없이 사용할 수 있어야 한다. 관리자 화면은 태블릿과 데스크톱에서도 통계와 표가 잘 보이도록 확장한다.

지원 기준:

- 360px 이상 작은 모바일 화면 지원
- 390px ~ 430px 일반 모바일 화면 최적화
- 768px 이상 태블릿 화면 지원
- 1024px 이상 데스크톱 화면 지원

모바일 화면:

- 첫 화면의 `부모로 참여하기`와 `교사로 참여하기`는 세로로 배치한다.
- 주요 버튼은 엄지로 누르기 쉬운 크기로 만든다.
- 기도문 PNG 3장은 화면 폭에 맞춰 세로 스크롤로 보여준다.
- 보석 수집장은 20칸이 한눈에 들어오도록 4열 x 5행 또는 화면 상황에 맞는 안정적인 그리드를 사용한다.
- 보석 팝업과 수집 애니메이션은 화면 중앙에 크게 보여준다.

태블릿 화면:

- 홈 화면과 수집장은 중앙 고정 폭 레이아웃을 유지한다.
- 보석 수집장은 더 넓은 여백과 큰 보석 크기를 사용할 수 있다.
- 관리자 대시보드는 요약 카드를 2열 이상으로 배치한다.

데스크톱 화면:

- 사용자 화면은 지나치게 넓어지지 않도록 중앙 최대 폭을 둔다.
- 관리자 화면은 통계 카드와 참여자 표를 넓은 화면에 맞게 확장한다.
- 부모 통계, 교사 통계, 명단 외 참여자 목록을 표 형태로 보기 좋게 제공한다.

반응형 구현 원칙:

- 글자가 버튼이나 카드 밖으로 넘치지 않아야 한다.
- 기도문 이미지는 원본 비율을 유지한다.
- 고정된 보석 칸, 버튼, 통계 카드가 데이터 변화로 흔들리지 않도록 안정적인 크기를 둔다.
- 모바일에서 가로 스크롤이 생기지 않게 한다.
- 관리자 표는 좁은 화면에서는 카드형 목록으로 전환한다.

## 15. 예외 상황과 처리

기도문 이미지가 3장 모두 없을 때:

- 사용자에게 `오늘 기도문을 준비 중입니다.` 표시
- 관리자에게는 업로드 누락 표시

기도문 일부만 업로드됐을 때:

- 관리자 화면에서 미완료 상태 표시
- 사용자에게는 가능한 한 3장 모두 준비된 후 공개하는 것을 권장

같은 날짜를 여러 번 완료할 때:

- 기존 완료 기록을 유지
- 중복 보석은 생성하지 않음

휴대폰을 바꾼 경우:

- 사용자는 다시 부모/교사 등록 흐름을 탈 수 있다.
- 동일 이름의 중복 참여자가 생길 수 있으므로 관리자 화면에서 중복 후보를 확인할 수 있으면 좋다.

명단 외 자녀 입력:

- 기능은 동일하게 제공
- 공식 반 통계에는 포함하지 않음
- 관리자 화면에서 별도 목록으로 표시

미래 날짜 빈 보석 클릭:

- 기도문으로 이동하지 않음
- `기도문은 당일 자정에 열립니다.` 안내

공유 이미지 생성 실패:

- 다시 시도 버튼
- 문구 복사 fallback

Web Share API 미지원:

- 이미지 저장
- 문구 복사
- 사용자가 카톡에 직접 붙여넣을 수 있게 안내
